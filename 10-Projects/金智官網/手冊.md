## 2. 怎麼上一篇 SEO 文章

1. 進 Notion「SEO 文章管理」DB → 新增一列
2. 必填 12 欄位：
   - 文章標題（Title） / Slug（URL 後綴，純英數連字號）
   - 內容類別（📝 部落格／🔧 案例／🛠 服務頁）
   - 關鍵字類別（🔴 商業／🟡 痛點／🟠 檢修申報／🔵 地區）
   - 目標客群（🟢 業主／🏠 管委會）
   - 目標關鍵字 / 字數
   - SEO Title（≤ 60 字）/ SEO Description（≤ 160 字）
   - 封面圖（1200×630 上傳即可，build 時會自動下載到 `public/images/notion/`）
   - 作者（一律 `郭金智｜智廷工程行負責人｜20+ 年消防工程實戰經驗`）
   - 上稿日期
3. 填內文（Notion blocks，支援 H2/H3、清單、引用、圖、表）
4. 「狀態」改成 **已上稿**（草稿狀態不會出公網）
5. 按頁面上方 **🚀 立刻發布** 按鈕 → 打 webhook → ISR 重建
6. 5–10 秒後 `https://kingzhi-web.vercel.app/knowledge/<slug>` 上線
7. 想暫存：「狀態」維持「草稿」即可

> ⚠️ **不要把作者欄位寫成「消防設備師證照加持」** — 客戶定位是消防工程廠商，不是設備師（SPEC AC-18 紅線）

## 4. 環境變數（Vercel Production）

| Key | 用途 |
| --- | --- |
| `NOTION_TOKEN` | Notion integration secret，讀 SEO DB |
| `NOTION_DATABASE_ID` | SEO 文章管理 DB ID |
| `REVALIDATE_SECRET` | Notion 🚀 按鈕 webhook header 驗證 |
| `RESEND_API_KEY` | 寄信 |
| `CONTACT_EMAIL_TO` | 詢問表單收件信箱 |
| `LINE_CHANNEL_ACCESS_TOKEN` | LINE Messaging API |
| `LINE_TARGET_GROUP_ID` | 「智廷詢價通知」群組 ID |
| `NEXT_PUBLIC_SITE_URL` | 站上各處組 URL 用 |
| `NEXT_PUBLIC_GA_ID` | GA4（待設） |

**本機 dev 拉 env：**

```bash
cd D:\GOLD\kingzhi-web
npx vercel env pull .env.local --environment=production
```

⚠️ Vercel 把所有 secret 變數都標 **Sensitive**，pull 下來會是 `""` 空字串。要本機跑真實 Notion 串接的話，得從 Notion integration 設定頁手動拷貝 token 進 `.env.local`。否則就走 `lib/mock-data.ts` 的 8 篇 fallback。
