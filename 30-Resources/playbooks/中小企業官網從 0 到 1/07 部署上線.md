# 07 部署上線｜git workflow + Vercel + GSC + 商家檔案

## Git workflow（小 PR 為王）

### 原則

**一個 PR = 一個動詞 + 一個對象**

- ✅ `feat: 加 hero 輪播`
- ✅ `chore: 升級 Next.js 14.2.35`
- ✅ `fix: 修 LINE button 假錨點`
- ❌ `feat: 多項改進`（範圍不明、不好 review）

### 標準操作

```bash
cd /d/GOLD/your-project
git fetch origin
git checkout -b feat/xxx origin/main   # 從最新 main branch off
# 做事
git add specific-files                  # 不要 -A
git commit -m "..."
git push -u origin feat/xxx
gh pr create --base main --head feat/xxx --title "..." --body "..."
# 合 PR → 自動部署
git checkout main && git pull
```

### Vercel Hobby + private repo 雷區

Vercel Hobby 會 enforce git commit author email **必須對齊** Vercel 帳號 email。

- 全域 git config 用 `xxx@gmail.com`，但 Vercel 帳號是 `yyy@gmail.com`
- → push 上去 Vercel 會擋 build，error 訊息超不直觀

**對策：** 該專案 repo 用 `git config user.email yyy@gmail.com` 覆蓋全域。

## Notion 內容上線（不需 code change）

```
1. Notion DB 編輯內容
2. 狀態 = 已上稿
3. 按頁面右上「🚀 立刻發布」button
4. 觸發 /api/revalidate webhook
5. ISR 重生該頁面（5–10 秒）
6. 上線
```

**體驗：** 業主只動 Notion，不用碰程式。

## Search Console（必做）

```
1. https://search.google.com/search-console
2. 「新增資源」→ 「URL 前置字元」（不是網域）
3. URL 結尾要含 /（例 https://kingzhi-web.vercel.app/）
4. 驗證方式：「HTML 標籤」
5. 複製 content="xxx"，加到 app/layout.tsx 的 metadata.verification.google
6. PR 合並後 Vercel 重 build
7. 回 GSC 按「驗證」綠勾
8. 左側選單「Sitemaps」→ 提交 sitemap.xml
9. 等 1–7 天 Google 開始爬
```

## Google 商家檔案（在地 SEO 核武）

特別針對地區性業者：

```
1. https://business.google.com
2. 加商家 → 商家類別（消防 / 水電 / 室內設計 / 美容）
3. 地址：注意隱私
   - 住商混合 → 勾「服務地區」、不顯示精確地址
   - 純商業地址 → 顯示完整地址
4. 服務地區：選縣市
5. 電話：對外公開那支
6. 網站：你的 URL
7. Google 寄明信片到登記地址（3–14 天）
8. 收到後輸驗證碼
9. 後續每週優化：加照片、回評論、發 Google 貼文
```

對在地服務業（小範圍、靠地理）影響：**比 SEO 文章還大**。

## 網域選擇

### 確認持有狀況

```bash
whois example.com   # 查註冊商 / 註冊日 / 到期日
```

紅燈訊號：
- 註冊日比客戶公司早很多 → 可能是別人的
- Name servers 在阿里雲（hichina）/ GoDaddy → 國外註冊
- 註冊商不在台灣 → 直接買新的可能比交涉快

### 註冊優先序

| 域名 | 適用 | 註冊商建議 |
| --- | --- | --- |
| `.com` | 國際取向 / 強品牌 | Cloudflare / Gandi |
| `.com.tw` | 商業正規 | HiNet / 戰國策 |
| `.tw` | 短、便宜、SEO 略加分 | HiNet 戰國策（TWNIC 代理） |

### 接到 Vercel

```
1. Vercel Dashboard → Project → Settings → Domains → Add
2. 註冊商後台改 Name Servers 為:
   ns1.vercel-dns.com
   ns2.vercel-dns.com
3. 等 15 分鐘 – 24 小時 DNS 傳播
4. 改 NEXT_PUBLIC_SITE_URL → 重 deploy
5. （optional）在 next.config.js 加 301 redirect 把舊 *.vercel.app → 新網域
```

## Resend 域名驗證（信件用自家域名）

```
1. https://resend.com → Domains → Add
2. Resend 給 3 個 DNS records（SPF / DKIM / DMARC）
3. 進註冊商後台加這 3 個 records
4. 等 Resend 後台變綠勾（5 分鐘 – 24 小時）
5. /api/contact 的 from 改成 noreply@yourdomain.com
```

驗證前的 from 用 `onboarding@resend.dev`（會被 Gmail 標 spam，但能用）。

## 上線後一週的 checklist

- [ ] Search Console 驗證 + 提交 sitemap
- [ ] Google 商家檔案申請（等明信片 14 天）
- [ ] 接正式網域到 Vercel
- [ ] Resend 域名驗證 SPF/DKIM
- [ ] GA4 設好，至少看到第一個 user
- [ ] Lighthouse 4 指標 ≥ 90
- [ ] 跨裝置測（手機 / iPad）
- [ ] 把官網連結放進現有 IG / FB / LINE 個人簽名

## 上線後一個月的 checklist

- [ ] Search Console 看「索引」報告，有幾頁被收
- [ ] GA4 / Vercel Analytics 看流量來源
- [ ] 商家檔案明信片到了、驗證完
- [ ] 至少加 3 篇新 SEO 文章
- [ ] 一次 Lighthouse 健檢

## 上線後三個月

- 累計 20+ 篇知識文章
- 第一筆 Google 自然流量
- 商家檔案有 1–3 則評論
- 第一筆從網站來的真實詢價
