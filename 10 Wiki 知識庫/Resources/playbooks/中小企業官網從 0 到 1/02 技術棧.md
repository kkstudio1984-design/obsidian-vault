# 02 技術棧｜為什麼這樣選

## 結論先講

**Next.js 14 App Router + Tailwind + Notion CMS + Vercel**

```
前端框架: Next.js 14 (App Router)
樣式:    Tailwind CSS + 自寫 CSS variables
CMS:     Notion DB（headless）
託管:    Vercel Hobby
版控:    GitHub private repo
分析:    GA4 + Meta Pixel（env-driven）
信件:    Resend
通訊:    LINE Messaging API
```

## 為什麼 Next.js（不是 WordPress / Wix）

| 標準 | Next.js | WordPress | Wix |
| --- | --- | --- | --- |
| SEO 控制 | 完整自訂 metadata / schema | 套件難客製 | 鎖死 |
| 編輯體驗 | Notion CMS（爽） | wp-admin（醜慢） | 拖拉 |
| 客製深度 | 任意 | plugin 撞 | 0 |
| 維運 | 自己負責但 Vercel 幫一半 | 自己負責 100% | 別人負責 |
| 月成本 | $0–$20 | $5–$30 + host | $14+ |
| 開發速度（熟手） | 1 天起 | 3 天 | 數小時但有限制 |

> 給**有開發能力的 1 人團隊**最佳。完全外行就 Wix，純內容站就 WordPress。

## 為什麼 Notion CMS（不是自架後台）

| 標準 | Notion | 自架（Strapi/Payload） |
| --- | --- | --- |
| 學習曲線 | 業主已會 | 業主要重新學 |
| 多平台同步 | 全平台原生 | 自己處理 |
| 範本 / 結構 | 拖拉建 DB | 寫 schema |
| 編輯時富文本 | 好用 | 看實作 |
| API 速度 | 慢（300–1000ms） | 快 |
| 廠商風險 | 高（鎖在 Notion） | 低 |

**取捨：** 業主編輯體驗 > API 速度。慢可以用 ISR + 本地圖片管線解決。

### Notion 的雷區（要對 client 設期待）

- API 速度慢，**build time** 抓料超過 50 篇可能要 60 秒
- 圖片用 S3 signed URL，**1 小時過期** → 必須 build time 下載到本地
- 多媒體（影片）不太行
- 多人協作 50+ 編輯者會卡
- API 配額：每秒 3 req（小站 OK）

## 為什麼 Vercel（不是 Cloudflare Pages / Netlify）

| 標準 | Vercel | CF Pages | Netlify |
| --- | --- | --- | --- |
| Next.js 整合 | 原生（同一公司） | 需 next-on-pages | OK |
| ISR | 完整 | 部分 | 完整 |
| 部署速度 | 最快 | 快 | 慢 |
| 免費額度 | 100GB bandwidth/mo | 無限 | 100GB |

Hobby 免費對小站夠用。流量超 Pro $20/mo。

## Notion CMS 配置（給 Next.js 吃）

### 必要環境變數

```
NOTION_TOKEN           # integration secret，ntn_...
NOTION_DATABASE_ID     # SEO DB 的 ID
REVALIDATE_SECRET      # webhook 用
NEXT_PUBLIC_SITE_URL   # 全站絕對 URL
RESEND_API_KEY         # 寄信
CONTACT_EMAIL_TO       # 收信信箱
LINE_CHANNEL_ACCESS_TOKEN  # LINE Bot
LINE_TARGET_GROUP_ID   # LINE 通知群組 ID
NEXT_PUBLIC_GA_ID      # GA4
NEXT_PUBLIC_FB_PIXEL_ID # Meta Pixel
```

### Integration 權限

- 給**整個 Notion 父頁面** + 加 「kingzhi-web」connection
- 如果新版 multi-source DB → 內嵌 DB 要**單獨**加 connection（不繼承父頁）

### 注意

- Notion 新 API 版本 `Notion-Version: 2025-09-03` 支援 data source
- `@notionhq/client` SDK 預設用舊版，要新 DB 要用 raw fetch

```json
{
  "dependencies": {

    "notion-to-md": "^3.1.7",
    "react-hot-toast": "^2.4.1",
    "react-markdown": "^9.0.1",
    "remark-gfm": "^4.0.0",
    "resend": "^4.0.1",
    "zod": "^3.23.8",
    "dayjs": "^1.11.13",
    "p-limit": "^6.1.0"
  }

```

關鍵套件功能：
- `notion-to-md` — Notion blocks → markdown（網站渲染用）
- `@tryfabric/martian` — markdown → Notion blocks（推內容用，npm install --no-save）
- `p-limit` — 控制 Notion 圖片下載並發數（避免 build 超時）
- `react-hot-toast` — 表單成功訊息
- `zod` — API 表單驗證
- `resend` — 信件發送

## 部署設定

### Vercel project

- Framework preset: Next.js（自動偵測）
- Root directory: `/`
- Build command: `npm run build`（預設）

### GitHub ↔ Vercel 自動部署

`git push origin main` → 約 40 秒上線

**注意：** Vercel Hobby + private repo 會 enforce git commit author 對齊 Vercel 帳號 email。第一次 commit 失敗看 author email 是否 match。
