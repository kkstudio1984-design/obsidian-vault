# 04 視覺資產｜AI 生圖 + 三層 fallback

## 為什麼不用實景照

| 實景照 | AI 生圖 |
| --- | --- |
| 真實感最強 | 一致風格 |
| 要客戶準備（拖延風險高） | 立刻產 |
| 不同照片風格不一 | 全站視覺統一 |
| 法律乾淨（自家拍） | 商用授權需挑工具 |
| 解析度可能不夠 | 預設 1200×630 對齊 OG |

**選 AI 生圖**：B2B 小站對「實景真實感」需求低，「視覺一致」與「上線速度」需求高。

## 工具選擇

| 工具 | 風格 | 商用 | 建議 |
| --- | --- | --- | --- |
| **Midjourney v6/v7** | 紀錄片質感最強 | Pro $30/mo | ⭐ Hero / 案例頁 |
| **DALL-E 3** (ChatGPT) | 中文 prompt 友善、平 | Plus $20/mo | ⭐ 量產文章封面 |
| **Adobe Firefly** | 乾淨設計感、最商用乾淨 | Creative Cloud | 客戶要 100% 商用授權時 |
| **Flux** | 寫實 | 依平台 | 跳過 |

> 金智案例：絕大多數用 DALL-E 3（時間效率），Hero / 案例頁混 MJ。

## 三層構圖原則（v2 動感版）

避免「乾淨無感的合成圖」必殺技。每張圖三層：

1. **前景 hero**：1 個服務主題物件（撒水頭、文件、設備）
2. **中景敘事道具**：3–5 個「正在發生這件事」的小物件（咖啡、工具、便條）
3. **背景環境**：景深虛化但有可辨識輪廓（廠房、辦公室、街景）
4. **光線氛圍**：暖金重點光 + 深黑陰影（如果品牌是黑金）
5. **動感**：mid-action 瞬間（壓力錶飆升、燈條測試亮起、紙張被風翻起）

### Prompt 範本

```
Editorial [composition style] of [scene], hero subject: [main object 詳細描述];
second focal point: [story 第二焦點];
mid-ground props: [3-5 道具列舉];
background: shallow-focus [深景描述];
lighting: [warm gold 主光] + [cool blue 副光] contrast;
mood: [氛圍描述];
[攝影風格] [焦距] no people, --ar 1.91:1 --style raw
Negative: legible text, brand logos, cartoon, people, hands, faces, watermark
```

關鍵 negative：
- `no readable text` — AI 寫文字 90% 機率拼錯
- `no people / no faces / no hands` — 真人臉問題、肢體扭曲
- `no brand logos` — 商標可能侵權

| 用途 | 規格 | 比例 | 位置 |

## fallback chain（極關鍵）

`lib/notion.ts` 三層 fallback：

```ts
const slugFallback = slug ?
  contentType === '🛠 服務頁'    ? `/images/services/${slug}.png` :
  contentType === '🔧 案例／實戰' ? `/images/cases/${slug}.png` :
  contentType === '📝 部落格文章' ? `/images/knowledge/${slug}.png` :
  null
: null;

const coverImage =
  notionCover ??           // 1. Notion 上傳的（業主可隨時換）
  slugFallback ??          // 2. 程式端 slug 對應圖（dev 期間 + 沒上傳時）
  '/images/og-default.jpg'; // 3. 統一退路
```

**效益：**
- dev 期間 Notion 不接還能看圖
- 業主上傳新圖立即蓋過程式 fallback
- 出大事至少有 og-default

## 工作流（從 prompt 到上線）

```
1. 寫 prompt（中文情境 + 英文 prompt 雙語）→ Notion 集中管理
2. AI 工具生成（1200×630 或 16:9）
3. 圖檔丟下載資料夾
4. 命名為 <slug>.png
5. cp 到 public/images/<type>/
6. 同時 cp 一份 _orig 到外部 assets 資料夾備份
7. commit + push
```

### 為什麼 assets 資料夾在 repo 外

`D:\GOLD\kingzhi-assets\` 放原檔（含 `_orig` 後綴），repo 內 `public/images/` 放實際用的圖。

- repo 不收原檔：避免 git 大檔（單張 2MB+ × 數十張）
- 原檔保留：未來改尺寸、換比例不用重新產
- 結構文件化：assets 資料夾內有 README 對應表

## og-default.jpg 設計

**最該細細打磨的單一張圖**。所有沒設封面的文章 fallback 到這。要：

- 含「核心訊息」slogan（例「20+ 年消防工程實戰」）
- 配色對齊品牌主色
- 純視覺、無關鍵物件（任何文章都不違和）
- 預留空間給 OG 平台疊文字（FB / LINE 偶爾加層）

> 金智案例：用了黑金 bokeh + 金色幾何裝飾 + 預留 logo 區。
