# Obsidian Web Clipper 作為 LLM Wiki 輸入層

## 定位

Obsidian Web Clipper 是 LLM Wiki 的低摩擦輸入工具。它負責把網頁、文章、選取文字、highlight 和影片頁面快速送進 vault，但不負責最終整理。

在本 vault 中，Web Clipper 的標準位置是：

```text
00 Inbox 原始資料/Web Clipper
```

## 為什麼不要直接剪到 Wiki

剪藏內容仍是原始資料，通常還沒經過判斷、提煉和連結。

如果直接進 Wiki，知識庫會變成新的收藏夾。正確流程是：

```text
Web Clipper -> Inbox -> Codex 整理 -> Wiki / Skills / Outputs
```

## 常用情境

1. 學習筆記：剪教學文，整理成可複習材料。
2. 主題研究：剪多篇同主題資料，做比較分析。
3. 內容創作：剪金句、案例、數據，做文章或腳本素材。
4. 專案資料：剪競品、案例、官方文件，回流到專案。
5. 靈感收集：剪短觀點，週末再整理。

## Template 設定提醒

Web Clipper 的 Vault 欄位要填 Obsidian 的 vault 名稱，不是 Windows 路徑。

本 vault 的名稱應使用：

```text
Obsidian-vault
```

不要填：

```text
D:\Obsidian-vault
```

Folder / Note location 則填：

```text
00 Inbox 原始資料/Web Clipper
```

## 每週整理指令

```text
請整理「00 Inbox 原始資料/Web Clipper」裡尚未整理的剪藏資料。

請列出每篇資料的主題、價值、建議去向，並判斷應該整理到 Wiki、沉澱成 Skill、產出 Output，或封存。
```
