# 每週知識庫維護節奏

來源：[[原始資料 03 - 個人知識庫工作流靈感筆記]]

## 核心概念

個人知識庫不需要每天深度整理。更穩定的節奏是：平日低摩擦收集，週末集中整理，週日複盤回流。

```text
平日收集 -> 週六整理 -> 週日複盤 -> 下週再收集
```

## 週一到週五：低摩擦收集

只做一件事：把可能有價值的資料放進 `00 Inbox 原始資料`。

可以收集 Web Clipper 剪藏、文章與影片、會議紀錄、專案資料、靈感片段和提示詞。

重點是降低收集成本，不在當下過度分類。

## 週六：集中整理

用 [[Inbox 資料整理 Skill]] 或 [[知識庫營運總流程 Skill]] 掃描 Inbox。

整理目標：找出值得留下的資料、更新或建立 Wiki 筆記、抽出可重複使用的 Skill、標記可以變成 Output 的內容、封存低價值或用途不明的檔案。

## 週日：複盤回流

用 [[每週知識庫複盤 Skill]] 檢查整套系統。

複盤問題：這週新增了哪些 Input？哪些資料已經進入 Wiki？哪些流程可以變成 Skill？哪些 Outputs 可以回流？下週最值得處理的 3 件事是什麼？

## 哪些事交給 AI

適合交給 AI：掃描 Inbox、摘要資料、初步分類、建議去向、產生複盤草稿、找出可沉澱的 Skill。

需要人決定：哪些資料真的重要、哪些主題值得長期追蹤、哪些內容該刪除或封存、哪些輸出符合自己的目標。

## 最小週循環

每週至少完成：清一次 Inbox、新增或更新至少一篇 Wiki、改進或新增一個 Skill、寫一份週複盤。
