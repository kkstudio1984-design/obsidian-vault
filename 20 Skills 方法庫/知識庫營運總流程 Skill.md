# 知識庫營運總流程 Skill

## 用途

這是目前 Obsidian LLM Wiki 的主流程 Skill，用來統一處理 Inbox 整理、Wiki 沉澱、Skill 抽取、Output 回流與週複盤。

它整併了：

- [[自生長知識庫整理流程]]
- [[Inbox 資料整理 Skill]]
- [[每週知識庫複盤 Skill]]

## 適用時機

- 每週整理 Obsidian vault。
- Web Clipper 剪藏累積後需要消化。
- 想把輸出成果回流成知識。
- 想檢查知識庫是否越來越乾淨、越來越可用。

## 主流程

```text
1. 收集：掃描 00 Inbox 原始資料
2. 篩選：判斷整理、延後、封存
3. 沉澱：建立或更新 10 Wiki 知識庫
4. 方法化：抽取 20 Skills 方法庫
5. 輸出：建立或更新 30 Outputs 輸出
6. 複盤：寫入 40 Reviews 複盤
7. 索引：更新 90 Index 索引
```

## 一鍵指令模板

```text
請幫我營運這週的 Obsidian LLM Wiki。

請依序完成：

1. 檢查「00 Inbox 原始資料」與「00 Inbox 原始資料/Web Clipper」有哪些尚未整理的內容。
2. 判斷每份資料的去向：整理到 Wiki、沉澱成 Skill、產出 Output、延後或封存。
3. 將成熟概念整併到「10 Wiki 知識庫」，避免重複筆記。
4. 將重複三次以上的流程整併到「20 Skills 方法庫」。
5. 檢查「30 Outputs 輸出」中有哪些成果可以回流成知識或 Skill。
6. 在「40 Reviews 複盤」建立本週複盤筆記。
7. 更新「90 Index 索引」的常用入口。

請採取保守整併：不要刪除原始資料；若要移動，優先移到 Archives 或 Inbox 的來源資料夾。
```

## 判斷規則

### 進 Wiki

有長期價值的概念、原理、比較或來源摘要；可被未來多次引用；能補強既有主題。

### 進 Skills

是可重複使用的流程、提示詞、SOP、檢查清單；已經或預期會重複使用三次以上。

### 進 Outputs

可以對外發布、交付或展示，例如文章、課程、報告、簡報、專案文件或腳本。

### 進 Archives

測試檔、重複檔、用途不明但暫時不刪的資料；已整理過但需要保留來源的剪藏。

## 每週交付物

每週跑完後，至少產出：

- 一份週複盤
- 一份新增或更新的 Wiki
- 一份新增或更新的 Skill
- 一個下週優先任務列表

## 注意事項

- 原始資料保留來源，不直接覆蓋。
- Wiki 要寫成可讀內容，不只是摘要。
- Skill 要能直接複用，不收一次性資訊。
- Output 要回流，不要和 Wiki 斷開。
