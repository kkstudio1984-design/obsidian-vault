# 每週知識庫複盤 Skill

## 用途

每週檢查 Obsidian vault 是否形成有效的知識循環。

框架：

```text
Input -> Filter -> Storage -> System -> Output
```

對應目前資料夾：

```text
00 Inbox 原始資料 -> 篩選判斷 -> 10 Wiki 知識庫 -> 20 Skills 方法庫 -> 30 Outputs 輸出 -> 40 Reviews 複盤
```

## 指令模板

```text
請幫我複盤這週的 Obsidian vault。

請檢查：
1. 00 Inbox 原始資料中有哪些尚未整理的內容。
2. 哪些資料值得整理到 10 Wiki 知識庫。
3. 哪些流程可以沉澱成 20 Skills 方法庫。
4. 30 Outputs 輸出中有哪些成果可以回流成知識。
5. 請在 40 Reviews 複盤 中建立本週複盤筆記。

複盤筆記請包含：
- Current State
- Input
- Filter
- Storage
- System
- Output
- Next Actions
```

## 複盤問題

### Input

- 這週新增了哪些資料？
- 哪些資料還沒整理？
- 哪些資料命名不清或需要封存？

### Filter

- 哪些資料值得深入？
- 哪些資料只是暫存？
- 哪些資料應該刪除、封存或延後？

### Storage

- 這週新增或更新了哪些 Wiki？
- 哪些概念需要合併？
- 哪些主題值得建立索引？

### System

- 哪些流程重複出現？
- 哪些流程可以變成 Skill？
- 哪些模板需要改進？

### Output

- 這週產出了哪些文件、課程、專案或報告？
- 哪些成果可以回流成知識？
- 下週最值得產出的 1-3 個成果是什麼？

## 輸出規則

- 結論要直接。
- 下一步要具體。
- 不要只寫感想，要指出可以搬移、建立、更新、封存的項目。
- 若知識庫還很薄，要明確說出下一批應該收集什麼。
