# 自生長知識庫整理流程

## 指令模板

請根據我的 Obsidian vault，整理最近新增到「00 Inbox 原始資料」的資料。

請完成以下任務：

1. 找出每份資料的主題、來源與核心觀點。
2. 將有價值的內容整理成「10 Wiki 知識庫」中的知識筆記。
3. 如果內容包含可重複使用的方法、流程、判斷標準或模板，請沉澱到「20 Skills 方法庫」。
4. 如果內容可以產出文章、報告、簡報或方案，請在「30 Outputs 輸出」建立輸出草稿。
5. 最後在「40 Reviews 複盤」新增一份複盤記錄，說明本次新增了哪些知識、哪些內容值得後續追蹤。

請保留來源連結或來源說明，並使用 Obsidian Markdown 格式。
